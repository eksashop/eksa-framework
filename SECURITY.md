# Kebijakan Keamanan

## Versi yang Didukung

Kami sangat menyarankan Anda untuk selalu menggunakan versi terbaru dari Eksa Framework untuk mendapatkan perbaikan keamanan terbaru.

| Versi | Didukung          |
| ------- | ------------------ |
| 3.5.x   | ✅ Ya               |
| < 3.5   | ❌ Tidak            |

## Melaporkan Kerentanan

Kami menangani keamanan dengan sangat serius. Jika Anda menemukan kerentanan keamanan di Eksa Framework, harap **JANGAN** membuka *Issue* publik. Sebaliknya, harap laporkan kerentanan tersebut secara privat melalui email ke:

**security@eksa-framework.dev** (Contoh email, silakan sesuaikan jika ada yang spesifik)

Tolong sertakan detail berikut dalam laporan Anda:
- Deskripsi kerentanan.
- Langkah-langkah untuk mereproduksi masalah.
- Dampak potensial dari kerentanan tersebut.

## Proses Respons

Setelah kami menerima laporan kerentanan:
1. Kami akan mengonfirmasi penerimaan laporan dalam waktu 48 jam.
2. Kami akan bekerja untuk memvalidasi dan memperbaiki masalah tersebut.
3. Kami akan memberikan pembaruan secara berkala mengenai kemajuan perbaikan.
4. Setelah perbaikan dirilis, kami akan memberikan kredit kepada pelapor (kecuali jika pelapor memilih untuk anonim).

Terima kasih telah membantu menjaga keamanan Eksa Framework!
